Github pages重新开始：使用Hugo、Typora (xujiahua.github.io)

安装Hugo

  • 下载地址：Releases · gohugoio/hugo

  • 配置环境变量

    检验：cmd输入

    E:\笔记>hugo version
hugo v0.89.2-63E3A5EB windows/amd64 BuildDate=2021-11-08T15:22:24Z VendorInfo=gohugoio

使用

  • 生成站点

    hugo new site e:/note

  • 创建文章

    $ cd e:/note
$ hugo new about.md	#创建博客

  • 安装皮肤

    # 创建 themes 目录
$ cd themes
$ git clone https://github.com/spf13/hyde.git

  • 运行Hugo

    hugo server

  • 部署

    hugo

    如果一切顺利，所有静态页面都会生成到 public 目录

发布到gitHub pages

将pubilc目录里所有文件 push 到刚创建的Repository的 main 分支。(我用的github desktop)

$ cd public
$ git init
$ git remote add origin https://github.com/coderzh/coderzh.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin main

浏览器里访问：http://coderzh.github.io/ （注意，如果看不到任何文章，请去掉文章头部的 draft=true 再重新生成。）

修改 E:\笔记\archetypes\default.md，去掉draft=true

图片不显示问题

  • 建立软连接

    #/d表示文件夹,E:\笔记\images为目标文件夹，E:\笔记\static\images为源文件夹
mklink /d E:\笔记\images E:\笔记\static\images

  • 设置typora的图片位置为：E:\笔记\images

我目前的文章在E:\笔记\content下，所以../images之后就成了E:\笔记\images。主要还是跟你的文章位置来设置typora的图片位置

在根目录建images软链接的理由
  1. hugo的静态资源放在static目录下
  2. hugo生成静态文件时，会将static目录下的东西平移到public目录下，比如static/images/1.png，平移后为images/1.png。
  3. hugo static目录下的资源目录不能是软链接，WTF。Unsupported symlink found https://github.com/gohugoio/hugo/issues/6225
  4. Typora可以用软链接。
